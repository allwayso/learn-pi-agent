### Abstract

虽然大型语言模型（LLMs）在语言理解和交互式决策等任务中展现了令人印象深刻的能力，但其推理（例如CoT，Chain-of-Thoughts）和行动（例如行动计划生成）的能力主要被作为独立主题进行研究。==在本文中，我们探索了以交错方式使用 LLMs 生成推理轨迹和任务特定行动的方法，从而在两者之间实现更大的协同作用：推理轨迹帮助模型诱导、跟踪和更新行动计划以及处理异常，而行动则允许其与外部来源（如知识库或环境）交互以收集额外信息。==我们将我们的方法命名为ReAct，并将其应用于一系列多样化的语言和决策任务，证明了其相对于最先进基线的有效性，以及相比缺乏推理或行动组件的方法在人类可解释性和可信度方面的提升。具体来说，在问答（HotpotQA）和事实验证（Fever）任务中，ReAct 通过与简单的 Wikipedia API 交互，克服了思维链推理中普遍存在的幻觉和错误传播问题，并生成了比无推理轨迹基线更可解释的类人任务解决轨迹。在两个交互式决策基准（ALFWorld和WebShop）上，ReAct在仅使用一两个上下文示例进行提示的情况下，分别以34%和10%的绝对成功率优于模仿学习和强化学习方法。项目网站及代码：[this https URL](https://react-lm.github.io/)

### Background

在 ReAct 之前，LLM 的应用主要沿两个独立方向发展。

一是 **推理**，以 CoT 为代表——模型基于内部表征生成思考链条，但整个过程是静态黑箱，无法与外部世界交互，容易产生事实幻觉和错误传播。

二是 **行动**，以具身任务中的动作生成为代表——将多模态环境用文本描述后让 LLM 输出行动计划，但没有利用 LLM 进行高层抽象推理，也不维护工作记忆。

简言之，只有推理没有行动会产生幻觉，只有行动没有推理则难以保持高层目标和追踪子任务进度，ReAct 的目标就是桥接这两者。

### Method

ReAct 的核心思想是将智能体的动作空间从 A（外部动作）扩展为 **Â = A ∪ L**，其中 L 是语言空间，用于容纳 thought（推理轨迹）。Thought 不直接影响外部环境，而是对当前上下文进行推理、整合信息并更新上下文，为后续推理或行动提供支撑。这形成了推理与行动的双向协同：thought 帮助模型诱导和调整行动计划、处理异常，action 则从外部环境获取信息反哺推理。

ReAct 具有四个显著特性：直观易设计（人类只需在动作旁记录思考过程）、通用灵活（适用于 QA、事实验证、文字游戏、网页导航等多种任务）、高性能且鲁棒（1-6 个 few-shot 示例即可泛化），以及人类对齐可控（可通过编辑 thought 实时修正 agent 行为而无需重训练）。

以 HotpotQA 为例，具体流程是 Thought-Action-Observation 循环：每步由模型先生成 Thought 推理当前状况，再生成 Action（search/lookup/finish），环境返回 Observation 后全部追加到 prompt 作为工作记忆，循环直到给出答案或达到步数上限。

### Major Finding

ReAct 最突出的价值在于 **消除幻觉**。论文对 HotpotQA 上的 200 条轨迹做了人工标注（表 2），发现 CoT 的失败案例中有 56% 来自幻觉——模型用内部知识编造了不存在的事实。而 ReAct 通过接入 Wikipedia API 获取真实信息，失败案例中的幻觉比例降到了 0%，成功案例中的假阳性也从 CoT 的 14% 降到 6%。但这是有代价的：ReAct 的结构约束（必须交错生成 thought 和 action）降低了推理的灵活性，导致推理错误率反而高于 CoT（47% vs 16%），常见表现是模型陷入重复生成相同 thought 和 action 的死循环。

这就引出了论文最重要的实践结论：**ReAct 和 CoT-SC 应该组合使用**，而非二选一。ReAct 擅长获取事实、避免幻觉，CoT 擅长构建推理结构。论文设计了两种组合策略——ReAct 在步数上限内未完成时回退到 CoT-SC，或 CoT-SC 多数投票不足半数时切换到 ReAct。结果是，组合方法仅用 3-5 个 CoT-SC 样本就超过了纯 CoT-SC 用 21 个样本的效果（HotpotQA EM 35.1, Fever Acc 64.6）。这揭示了一个更一般的道理：==合理结合模型内部知识与外部检索知识，远比单纯堆叠其中一种更高效。==

在决策任务上，ReAct 同样大幅领先：ALFWorld 最佳成功率 71%（Act 仅 45%，BUTLER 37%），WebShop 成功率 40%（Act 30.1%，IL+RL 28.7%）。一个值得注意的消融实验是，若将 ReAct 的稀疏推理替换为类似 Inner Monologue 的密集反馈式推理（ReAct-IM），成功率骤降至 53%——说明有价值的不是"有推理"，而是"在关键位置做高层目标分解和常识推理"。此外，ReAct 在 PaLM-8B/62B 上的提示效果最初是四种方法中最差的（小模型难以同时从示例中学会推理和行动），但仅用 3000 样本微调后就反转为最佳——微调教给模型的是"如何通过行动获取信息并推理"，而非死记硬背知识。

### Conclusion

ReAct 是第一个将推理和行动统一为一个交错生成范式的 work，其 Thought-Action-Observation 循环成为后续几乎所有 agent 框架的基本范式——从 LangChain、AutoGPT 到 OpenAI function calling、Claude tool use，都延续了"模型自主决定何时推理、何时调用工具、如何根据环境反馈调整计划"这一核心思路。

在它之前，推理（CoT）和工具使用（WebGPT）是两拨人在做两件事；在它之后，agent = LLM + reasoning + tool use 成为共识。论文中关于稀疏推理优于密集推理的消融，以及通过编辑 thought 实现人在回路可控性的思路，也影响了后续 agent 的交互设计。

当然，当时的 ReAct 仍依赖固定的 few-shot prompt 和有限的 Wikipedia API 动作空间，距离真正自主的 agent 还有距离，论文也坦承需要更多训练数据和与强化学习的结合来进一步释放 LLM 的潜力——这些方向在之后的两三年里几乎全部成为了 agent 研究的主流。
